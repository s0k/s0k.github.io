Git 学习

Git属于分布式版本控制系统（SVN属于集中式版本控制系统）

Git基础设置

设置签名（作用是区分每个人）

1.设置用户名

Git config --global user.name 's0k'

2.设置用户名邮箱

Git config --global user.email 's0k@qq.com'

设置的签名和github账号没关系（可以不一样）

签名级别

1.项目级别/仓库级别：仅在本地库范围内有效

git config user.name/email tom

2.系统用户级别：登录当前操作系统的用户范围

git config --global user.name/email tom

注： .git文件为隐藏文件，里面有config文件，储存着用户名和邮箱以及密码。

比如：url = git@github.com:s0k/s0k.github.io.git

Git 核心

Git 最核心的一个概念就是工作流。

  • 工作区(Workspace)是电脑中实际的目录。
  • 暂存区(Index)类似于缓存区域，临时保存你的改动。
  • 仓库区(Repository)，分为本地仓库和远程仓库。

创建版本库

​ 版本库又名仓库，英文名repository，可以简单理解成一个目录，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以"还原”。

git init

1.创建文件夹 mkdir test  #新建文件夹

2.在文件夹内初始化Git（创建Git仓库）

cd test

git init  #初始化（初始化后会在本地创建一个隐藏的.git文件，这个目录是Git来跟踪管理版本库的）

向仓库提交文件

1.创建a.py文件

touch a.py

git add从工作区提交到暂存区
git commit从暂存区提交到本地仓库
git push或git svn dcommit从本地仓库提交到远程仓库
文件流程：工作区 -- 暂存区 -- Git仓库

2.查看git状态（查看当前文件在哪个区域以及文件状态）

git status

3.提交工作区文件到暂存区

git add a.py  #git add 文件

git add -A #提交所有文件

注：

从暂存区撤回：git rm --cached a.py  #git rm –cached 文件

4.提交暂存区文件到Git区

git commit -m "add a.py"  #git commit -m “提交描述” （m message）

版本回溯

查看日志

git log

commit 2774013c5300d1ef949d35d527589bc247a2d342 (HEAD -> master)   
# 以这串数字为提交的hash值来标明值，HEAD 是指针 指明当前版本
Author: s0k <xxxx@qq.com>    #作者
Date:   Sat Feb 22 23:32:54 2020 +0800  #时间
    练习git   #备注

日志以 行 来显示

git log --pretty=oneline

简洁版以行显示

git log --oneline

在oneline的基础上显示到某个版本需要的步数

git reglog

HEAD@{x} #移动到当前版本需要移动多少步

显示方式 同linux中的less

空格 向下翻页

b 向上翻页

q 退出

分支

在版本控制过程中，使用多条线同时推进多个任务

master（主干）

新开辟分支：

feature_blue（前端模块）

feature_game（游戏模块）

最后合并分支

分支的好处：

同时并行推进多个功能开发，提高开发效率

各个分支在开发过程中，如果某一个分支开发失败，不会对其他分支有任何影响。失败的分支删除重新开始即可。

分支操作

git status 结果有master，默认主干为master

查看分支

git branch -v

添加分支

git branch hot_fix  #git branch 分支名

切换分支

git checkout hot_fix  #git checkout 分支名

git branch -v #发现已经在hot_fix分支

hot_fix修改好后进行合并分支

合并分支

1.切换到接受修改的分支（被合并，增加新内容）上

2.执行merge命令

git branch -v
git checkout master
git branch -v
git merge hot_fix

合并分支产生冲突

master和hot_fix 同时修改一个文件

提示手动修改

git status

解决冲突：

1.编辑文件，删除特殊符号

2.把文件修改到满意的程度

3.git add 文件名

4.git commit -m ‘日志信息’  #此时commit不要带文件名

修改仓库文件

1.修改a.py cat a.py  #编辑a.py

2.提交工作区文件到暂存区 git add a.py  #提交a.py

删除仓库文件

1.删除文件

rm -rf a.py  #rm -rf 文件

2.从git删除 git rm a.py  #git em 文件

3.暂存区到Git区 git commit -m  #删除文件

Git管理远程仓库

一个服务器，多人共同开发

本地仓库同步到Git远程仓库中

Git 克隆操作

目的：将远程仓库（GitHub对应的项目）复制到本地

git config --list #列出所有 Git 当时能找到的配置

代码：

git clone https://github.com/s0k/s0k.github.io.git #git clone 仓库地址

给本地远程库起别名

查看别名

get remote -v

添加别名

git remote add origin https://github.com/s0k/s0k.github.io.git

#origin 是远程库（后面的链接）的别名， add 添加

Git推送

推送 origin（远程库的别名）

git push origin master

远程库修改的拉取

pull = fetch + merge

抓取远程库文件

git fetch origin master   #git fetch 远程库地址 远程分支名

#远程合并到本地

git merge origin/master    #git merge 远程库地址/远程分支名

直接执行pull(条件 不会产生冲突的情况下)

git pull origin master

解决团队协作会冲突

要点：

1.如果不是基于GitHub远程库的最新版所做的修改，不能推送，必须先拉取。

2.拉取下来后如果进入冲突状态，则按照“分支冲突解决”操作解决即可。
